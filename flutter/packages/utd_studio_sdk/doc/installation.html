utd_studio_sdk · v0.1.0

Installing UTD Studio SDK

utd_studio_sdk is consumed as a path dependency. It carries a vendored fork of stac under vendor/; the host app wires the runtime ports once at boot via UtdStudio.init.

1 · Add the path dependency

In the app entrypoint pubspec.yaml (e.g. utd_app / chatPackageV2):

dependencies:
  utd_studio_sdk:
    path: packages/utd_studio_sdk   # adjust to your relative path

2 · Duplicate the dependency_overrides (the known footgun)

This is the #1 mistake. pub honours dependency_overrides only from the entrypoint (app) pubspec — a library package's overrides are ignored transitively. So the vendored-stac override block below MUST be present identically in every app that depends on utd_studio_sdk. If you forget it, pub fetches the real stac from pub.dev → you end up with two copies of the engine, StacParser type identity breaks, and parser / action registration silently no-ops (screens render blank or ignore your custom widgets, with no error).

Add to the app entrypoint pubspec.yaml (paths point into the SDK's vendor/):

dependency_overrides:
  stac:
    path: packages/utd_studio_sdk/vendor/stac
  stac_core:
    path: packages/utd_studio_sdk/vendor/stac_core
  stac_framework:
    path: packages/utd_studio_sdk/vendor/stac_framework
  stac_logger:
    path: packages/utd_studio_sdk/vendor/stac_logger

The same block lives in the SDK's own pubspec.yaml (so it analyzes standalone), but that copy does nothing for the app build — only the entrypoint's copy counts. A CI grep guards against stray package:stac/ imports outside vendor/.

Then resolve:

flutter pub get

3 · Initialize at boot

Call UtdStudio.init once before runApp. Implement the required ports (StacTransport over your HTTP client, KeyValueCache over Hive / SharedPreferences) and any optional ports for the core.* actions you use: 

import 'package:utd_studio_sdk/utd_studio_sdk.dart';

await UtdStudio.init(StudioConfig(
  transport: MyHttpTransport(),   // required — /stac, /stac/{name}, /stac/{name}/version
  cache:     MyKeyValueCache(),   // required — namespaced key/value store
  navigator: MyNavigator(),       // optional — core.navigate / core.back / core.openDialog
  theme:     MyThemeSource(),     // optional — core.toggleTheme
  locale:    MyLocaleSource(),    // optional — core.setLocale
  toast:     MyToastSink(),       // optional
  translate: (ctx, key) => ctx.tr(key), // optional — t.* / tKey i18n binding
  extraParsers: [/* custom StacParser widgets */],
  extraActions: [/* custom StacActionParser actions */],
));

init runs Stac.initialize once, opens the screen cache, and kicks a background syncAll() — so startup never blocks on the network.

4 · Render screens & register data

// Render a server-driven screen by name:
StacDynamicScreen(screenName: 'profile');

// Register data sources your screens bind to:
UtdStudio.registerObject('core.currentUser', () async => myUserMap);
UtdStudio.registerList('feed.items', () async => myItemsList);

Your backend must serve the screen contract: GET /stac, GET /stac/{name}, GET /stac/{name}/version.

Re-merging upstream

To bump the vendored Stac version, replace vendor/<pkg>/lib wholesale (incl. all generated .g.dart — never run build_runner against vendor/), update the override paths if a coordinate changed, then re-run flutter pub get + flutter analyze. Full runbook in UPSTREAM.md.

utd_studio_sdk · v0.1.0

تثبيت حزمة UTD Studio

تُستهلك utd_studio_sdk كـاعتماد بالمسار (path dependency). تحمل نسخة مُضمَّنة من stac تحت vendor/؛ ويربط التطبيق المضيف منافذ التشغيل مرة واحدة عند الإقلاع عبر UtdStudio.init.

١ · أضف الاعتماد بالمسار

في ملف pubspec.yaml الخاص بنقطة دخول التطبيق (مثل utd_app / chatPackageV2):

dependencies:
  utd_studio_sdk:
    path: packages/utd_studio_sdk   # عدّل المسار النسبي حسب مشروعك

٢ · كرّر كتلة dependency_overrides (المزلق المعروف)

هذا أكثر خطأ شيوعًا. يحترم pub كتلة dependency_overrides من pubspec نقطة الدخول (التطبيق) فقط — وتُتجاهَل تجاوزات الحزم المكتبية عبر الاعتماد. لذا يجب وجود كتلة تجاوز stac المُضمَّن أدناه بشكل مطابق في كل تطبيق يعتمد على utd_studio_sdk. وإن نسيتها، يجلب pub نسخة stac الحقيقية من pub.dev، فتحصل على نسختين من المحرّك، وتنكسر هوية نوع StacParser، ويفشل تسجيل المحلّلات والإجراءات بصمت (تظهر الشاشات فارغة أو تتجاهل عناصرك المخصّصة دون أي خطأ).

أضِف إلى pubspec.yaml الخاص بنقطة الدخول (تشير المسارات إلى vendor/ داخل الحزمة):

dependency_overrides:
  stac:
    path: packages/utd_studio_sdk/vendor/stac
  stac_core:
    path: packages/utd_studio_sdk/vendor/stac_core
  stac_framework:
    path: packages/utd_studio_sdk/vendor/stac_framework
  stac_logger:
    path: packages/utd_studio_sdk/vendor/stac_logger

توجد الكتلة نفسها في pubspec.yaml الخاص بالحزمة (لتُحلَّل بمفردها)، لكن تلك النسخة لا تفعل شيئًا لبناء التطبيق — نسخة نقطة الدخول وحدها هي التي تُحتسب. ويحرس فحص في CI ضد أي استيراد شارد لـpackage:stac/ خارج vendor/.

ثم نفّذ:

flutter pub get

٣ · التهيئة عند الإقلاع

استدعِ UtdStudio.init مرة واحدة قبل runApp. طبّق المنافذ المطلوبة (StacTransport فوق عميل HTTP، وKeyValueCache فوق Hive / SharedPreferences) وأي منافذ اختيارية لإجراءات core.* التي تستخدمها: 

import 'package:utd_studio_sdk/utd_studio_sdk.dart';

await UtdStudio.init(StudioConfig(
  transport: MyHttpTransport(),   // مطلوب — /stac, /stac/{name}, /stac/{name}/version
  cache:     MyKeyValueCache(),   // مطلوب — مخزن مفتاح/قيمة بنطاقات
  navigator: MyNavigator(),       // اختياري — core.navigate / core.back / core.openDialog
  theme:     MyThemeSource(),     // اختياري — core.toggleTheme
  locale:    MyLocaleSource(),    // اختياري — core.setLocale
  toast:     MyToastSink(),       // اختياري
  translate: (ctx, key) => ctx.tr(key), // اختياري — ربط ترجمة t.* / tKey
  extraParsers: [/* عناصر StacParser مخصّصة */],
  extraActions: [/* إجراءات StacActionParser مخصّصة */],
));

يُشغّل init الدالة Stac.initialize مرة واحدة، ويفتح كاش الشاشات، ويُطلق syncAll() في الخلفية، فلا يتوقف الإقلاع على الشبكة.

٤ · عرض الشاشات وتسجيل البيانات

// عرض شاشة موجَّهة من الخادم بالاسم:
StacDynamicScreen(screenName: 'profile');

// تسجيل مصادر البيانات التي تربط بها الشاشات:
UtdStudio.registerObject('core.currentUser', () async => myUserMap);
UtdStudio.registerList('feed.items', () async => myItemsList);

يجب أن يقدّم خادمك عقد الشاشات: GET /stac وGET /stac/{name} و GET /stac/{name}/version.

إعادة دمج النسخة الأصلية

لترقية نسخة Stac المُضمَّنة، استبدل vendor/<pkg>/lib بالكامل (بما في ذلك كل .g.dart المُولَّد — ولا تُشغّل build_runner على vendor/ أبدًا)، وحدّث مسارات التجاوز إن تغيّر أي إحداثي، ثم أعد تشغيل flutter pub get و flutter analyze. الدليل الكامل في UPSTREAM.md.
