Installation

Backend

  1. Place the package at backend/packages/profile.
  2. Register it as a path repository / require it in the app composer.json (package name utd/profile), then run composer update utd/profile. The Utd\Profile\Providers\ProfileServiceProvider is auto-discovered via the package's extra.laravel.providers.
  3. The provider extends BaseModuleServiceProvider, which auto-loads routes/api.php, migrations and translations, all gated by the package being enabled. It also contributes the Flutter profile.* UI strings to the backend translation catalog and registers its UTD Studio manifest.
  4. Enable the package from the admin dashboard under Settings → Packages (slug profile). Only when enabled does it load its routes, register the rich Filament infolist (taking over the user-view screen), and expose GET /api/users/{id}/profile.

The endpoint is protected by auth:sanctum plus the base ban / token / last-seen middleware, so no extra route wiring is needed.

Flutter

  1. Place the package at flutter/packages/profile and add it as a path dependency in the app pubspec.yaml: 
    dependencies:
  profile:
    path: packages/profile
    then run flutter pub get.
  2. Register the feature with the app's feature registry where the other features are wired: 
    import 'package:profile/profile.dart';

featureRegistry.register(ProfileFeature());
  3. That's it — ProfileFeature registers its routes, the mini_profile_card and self-profile ("Me") widgets, the profile.user Stac source, and bundles its profile.* translations. The feature is gated by the backend profile package: if the admin disables it, the app auto-disables the feature and the "Me" tab falls back to the base self-profile fallback.

التثبيت

الخلفية (Backend)

  1. ضع الحزمة في المسار backend/packages/profile.
  2. سجّلها كمستودع مسار (path repository) واطلبها في composer.json للتطبيق (اسم الحزمة utd/profile)، ثم نفّذ composer update utd/profile. يُكتشف المزوّد Utd\Profile\Providers\ProfileServiceProvider تلقائيًا عبر extra.laravel.providers.
  3. يرث المزوّد من BaseModuleServiceProvider الذي يحمّل تلقائيًا routes/api.php والهجرات والترجمات، وكلها مشروطة بتفعيل الحزمة. كما يساهم بسلاسل واجهة فلاتر profile.* في كتالوج ترجمات الخلفية ويسجّل بيان UTD Studio.
  4. فعّل الحزمة من لوحة الإدارة عبر الإعدادات ← الحِزم (Packages) (المعرّف profile). عند التفعيل فقط تُحمَّل مساراتها ويُسجَّل infolist الغني (فيستحوذ على شاشة عرض المستخدم) وتُتاح GET /api/users/{id}/profile.

نقطة النهاية محميّة بـ auth:sanctum إضافةً إلى وسطاء الحظر والتوكن وآخر ظهور في القاعدة، فلا حاجة لأي ربط إضافي للمسارات.

فلاتر (Flutter)

  1. ضع الحزمة في flutter/packages/profile وأضِفها كاعتماد مسار في pubspec.yaml للتطبيق: 
    dependencies:
  profile:
    path: packages/profile
    ثم نفّذ flutter pub get.
  2. سجّل الميزة في سجل ميزات التطبيق حيث تُربط الميزات الأخرى: 
    import 'package:profile/profile.dart';

featureRegistry.register(ProfileFeature());
  3. هذا كل شيء — يسجّل ProfileFeature مساراته وبطاقة mini_profile_card وودجت الملف الذاتي ("أنا"، Me) ومصدر Stac profile.user ويحزم ترجمات profile.*. وتُضبط الميزة عبر حزمة profile الخلفية: إن عطّلها المسؤول يتعطّل التطبيق تلقائيًا ويعود تبويب "أنا" إلى الملف الاحتياطي الأساسي.
