Reels

A short-video feed for the app — users record/upload vertical clips, scroll an endless ranked feed, and react, comment, view, gift, and report.

What it does

  • Short-video feed — vertical reels with caption, poster frame, and author. Filters: the default mixed feed, a following feed (when the Follow graph is available), per-user reels, and the current user's "my reels".
  • Weighted ranking + diversity — the feed serves a shared, briefly-cached "deck" of the most-recent reels. When ranking is on it is ordered by a recency-decayed engagement score (likes / comments / views) through a weighted shuffle plus an author-diversity pass, so fresher, better reels surface while no single author dominates. Each user is rotated to a different start offset in the deck so concurrent viewers read different windows instead of all hammering the newest few. A best-effort, cache-only seen-exclusion remembers what each user already watched and skips it on later pages and pull-to-refresh.
  • Likes & reactions — Facebook-style reactions (like, love, haha, wow, sad, angry). Toggle / switch / remove with a denormalized like_num counter and a per-type breakdown.
  • Comments — top-level comments and one level of replies, comment reactions, and a denormalized comment_num counter (replies count too; deleting a parent cascades its replies).
  • Views — atomic per-reel view counter (view_num).
  • Reports — users report reels and comments; reports land in an admin moderation queue (Filament resources) and notify admins.
  • Gifting — send a gift on a reel. The gift endpoint is always wired but degrades gracefully (HTTP 503) until the Gifts package binds App\Contracts\GiftSender; once installed, gifting auto-enables.

How it fits the platform

  • Standalone Composer package (utd/reels, namespace Utd\Reels) — a manual drop-in, not an nwidart module.
  • Depends on the Base only through contracts: MediaUploader (video upload), the notification system, and the optional GiftSender.
  • Poster frames are generated in-package via FFMpeg (pbmedia/laravel-ffmpeg), best-effort — a missing ffmpeg binary never fails reel creation.
  • Contributes a "Reels" tab to the Base user profile, Filament admin resources, and notification types (like / comment / comment-like / gift / report).
  • Toggle on/off from admin/packages; the route group is gated by package.enabled:reels.

الريلز (Reels)

خلاصة فيديوهات قصيرة للتطبيق — المستخدم بيصوّر/يرفع مقاطع رأسية، ويتصفّح فيد لا نهائي مرتّب، ويتفاعل، ويعلّق، ويشاهد، ويهدي، ويبلّغ.

بيعمل إيه

  • فيد فيديوهات قصيرة — ريلز رأسية مع وصف وصورة غلاف (poster) وصاحب المقطع. فلاتر: الفيد المختلط الافتراضي، وفيد following (لما يكون جراف المتابعة متاح)، وريلز مستخدم معيّن، وريلزي أنا.
  • ترتيب موزون + تنويع — الفيد بيقدّم "ديك" (deck) مشترك ومخزّن مؤقتًا لأحدث الريلز. لما الترتيب مفعّل بيتم ترتيبه حسب درجة تفاعل بتقل مع الوقت (لايكات / كومنتات / مشاهدات) عبر خلط موزون + تمريرة تنويع للمؤلّفين، فالأحدث والأفضل يطلع لفوق من غير ما مؤلّف واحد يسيطر. كل مستخدم بيتدوّر على نقطة بداية مختلفة في الديك عشان المتصفّحين المتزامنين يقروا نوافذ مختلفة بدل ما الكل يضرب على أحدث القليل. وفيه استبعاد للمُشاهَد (best-effort، في الكاش بس) بيفتكر اللي المستخدم شافه ويتخطّاه في الصفحات اللاحقة والسحب للتحديث.
  • لايكات وتفاعلات — تفاعلات بنمط فيسبوك (like, love, haha, wow, sad, angry). تبديل / تغيير / إزالة مع عدّاد مُجمّع like_num وتفصيل لكل نوع.
  • كومنتات — كومنتات رئيسية مع مستوى واحد من الردود، وتفاعلات على الكومنت، وعدّاد مُجمّع comment_num (الردود بتتعدّ كمان؛ حذف الكومنت الأصلي بيحذف ردوده).
  • مشاهدات — عدّاد مشاهدات ذرّي لكل ريل (view_num).
  • بلاغات — المستخدمون بيبلّغوا عن الريلز والكومنتات؛ البلاغات بتروح لطابور مراجعة في لوحة الأدمن (Filament) وبتبلّغ الأدمنز.
  • الهدايا — إرسال هدية على الريل. نقطة الهدية موصولة دايمًا لكن بتتعطّل بلطف (HTTP 503) لحد ما باكيج Gifts يربط App\Contracts\GiftSender؛ وبعد التثبيت بتتفعّل الهدايا أوتوماتيك.

مكانه في المنصّة

  • باكيج Composer مستقل (utd/reels، نيمسبيس Utd\Reels) — تثبيت يدوي drop-in، مش nwidart module.
  • بيعتمد على الـ Base عبر contracts بس: MediaUploader (رفع الفيديو)، ونظام الإشعارات، وGiftSender الاختياري.
  • صور الغلاف بتتولّد جوّه الباكيج عبر FFMpeg (pbmedia/laravel-ffmpeg)، best-effort — غياب بايناري ffmpeg مش بيفشّل إنشاء الريل أبدًا.
  • بيضيف تبويب "Reels" لبروفايل مستخدم الـ Base، وموارد أدمن Filament، وأنواع إشعارات (لايك / كومنت / لايك-كومنت / هدية / بلاغ).
  • تشغيل/إطفاء من admin/packages؛ مجموعة الراوتس محميّة بـ package.enabled:reels.
