Reels — Installation

The Reels feature has two halves: the backend Composer package (utd/reels) and the Flutter package (reels). Install both.

Backend

  1. Path — drop the package at backend/packages/reels/ (standalone Composer package, namespace Utd\Reels; manual drop-in, not auto-discovered).
  2. Autoload — add the PSR-4 entries to the root backend/composer.json: 
    "autoload": { "psr-4": { "Utd\\Reels\\": "packages/reels/src/" } }
"autoload-dev": { "psr-4": { "Utd\\Reels\\Tests\\": "packages/reels/tests/" } }
  3. Service provider — register Utd\Reels\Providers\ReelsServiceProvider (auto-discovered via the package's extra.laravel.providers, or add it by hand to config/app.php). The provider merges config, loads migrations / views / translations, registers routes, the profile tab, and the notification types, and self-registers with the PackageRegistry.
  4. Filament admin — add the plugin to AdminPanelProvider: 
    ->plugin(\Utd\Reels\Filament\ReelsPlugin::make())
    This exposes the reels feed, the reel resource, and the report (reel / comment) moderation queues.
  5. FFMpeg — the package requires pbmedia/laravel-ffmpeg (pulled in by Composer). The ffmpeg / ffprobe binaries must exist on the host for poster frames; if missing, reel creation still succeeds (no poster, best-effort).
  6. Install + migrate: 
    composer update          # pulls pbmedia/laravel-ffmpeg
composer dump-autoload
php artisan migrate
  7. Register the package so it shows in admin/packages and /api/packages/installed: 
    php artisan utd:sync-packages
  8. Enable in admin — open admin/packages and switch Reels on. The API route group is gated by package.enabled:reels and returns 403 while disabled.
Dev seed route. GET /api/reals/seed fills the feed with ~100 demo reels (ReelsBulkSeeder). It runs freely in local / development / testing; in any other environment it requires ?key= to match config('reels.seed_key') (set REELS_SEED_KEY), falling back to the baked-in default utd-reels-seed so it works on a config-cached server. Convenience only — do not rely on it in production.

Verify

php artisan route:list --path=reals
php vendor/bin/phpunit packages/reels/tests --no-coverage

Flutter

  1. Path dependency — the package lives at flutter/packages/reels/ (pubspec name reels). Add it to the host app's pubspec.yaml: 
    dependencies:
  reels:
    path: packages/reels
    then run flutter pub get.
  2. Register the feature — add ReelsFeature() to buildFeatures() in the host app's main.dart: 
    import 'package:reels/reels.dart';

List<AppFeature> buildFeatures() => [
  AuthFeature(),
  // ...
  ReelsFeature(),
];
    The feature self-wires its providers, routes, bottom-nav entry, profile tab, and translations. Server/admin overrides for the reels.* UI strings win automatically.

Enable/disable is honored at runtime: the app reads /api/packages/installed, and a disabled Reels package hides its UI contributions.

الريلز — التثبيت

ميزة الريلز ليها نصّين: باكيج الباك إند على Composer (utd/reels) وباكيج فلاتر (reels). ثبّت الاتنين.

الباك إند

  1. المسار — حُط الباكيج في backend/packages/reels/ (باكيج Composer مستقل، نيمسبيس Utd\Reels؛ تثبيت يدوي drop-in مش بيتكتشف أوتوماتيك).
  2. الـ autoload — ضيف مدخلات PSR-4 في backend/composer.json الرئيسي: 
    "autoload": { "psr-4": { "Utd\\Reels\\": "packages/reels/src/" } }
"autoload-dev": { "psr-4": { "Utd\\Reels\\Tests\\": "packages/reels/tests/" } }
  3. الـ Service Provider — سجّل Utd\Reels\Providers\ReelsServiceProvider (بيتكتشف عبر extra.laravel.providers بتاع الباكيج، أو ضيفه يدوي في config/app.php). الـ provider بيدمج الكونفيج، ويحمّل الـ migrations / views / الترجمات، ويسجّل الراوتس وتبويب البروفايل وأنواع الإشعارات، ويسجّل نفسه في الـ PackageRegistry.
  4. أدمن Filament — ضيف البلَجين في AdminPanelProvider: 
    ->plugin(\Utd\Reels\Filament\ReelsPlugin::make())
    ده بيظهّر فيد الريلز، ومورد الريل، وطوابير مراجعة البلاغات (ريل / كومنت).
  5. FFMpeg — الباكيج بيتطلّب pbmedia/laravel-ffmpeg (بتتسحب مع Composer). لازم بايناريز ffmpeg / ffprobe موجودة على السيرفر عشان صور الغلاف؛ ولو ناقصة، إنشاء الريل بينجح برضه (من غير غلاف، best-effort).
  6. التثبيت + الـ migrate: 
    composer update          # بتسحب pbmedia/laravel-ffmpeg
composer dump-autoload
php artisan migrate
  7. سجّل الباكيج عشان يبان في admin/packages و/api/packages/installed: 
    php artisan utd:sync-packages
  8. فعّل من الأدمن — افتح admin/packages وشغّل Reels. مجموعة راوتس الـ API محميّة بـ package.enabled:reels وبترجّع 403 وهي مطفيّة.
راوت الـ seed للتطوير. GET /api/reals/seed بيملا الفيد بحوالي 100 ريل تجريبي (ReelsBulkSeeder). بيشتغل بحرّية في local / development / testing؛ وفي أي بيئة تانية بيطلب ?key= يطابق config('reels.seed_key') (اضبط REELS_SEED_KEY)، مع fallback للقيمة الافتراضية المدمجة utd-reels-seed عشان يشتغل على سيرفر مكوّن بـ config cache. للراحة بس — متعتمدش عليه في الإنتاج.

التحقق

php artisan route:list --path=reals
php vendor/bin/phpunit packages/reels/tests --no-coverage

فلاتر

  1. اعتماد بالمسار (path dependency) — الباكيج موجود في flutter/packages/reels/ (اسم الـ pubspec هو reels). ضيفه في pubspec.yaml بتاع التطبيق المضيف: 
    dependencies:
  reels:
    path: packages/reels
    وبعدين شغّل flutter pub get.
  2. سجّل الميزة — ضيف ReelsFeature() في buildFeatures() جوّه main.dart بتاع التطبيق المضيف: 
    import 'package:reels/reels.dart';

List<AppFeature> buildFeatures() => [
  AuthFeature(),
  // ...
  ReelsFeature(),
];
    الميزة بتوصّل نفسها: الـ providers والراوتس وأيقونة الـ bottom-nav وتبويب البروفايل والترجمات. وتعديلات السيرفر/الأدمن لنصوص reels.* بتغلب أوتوماتيك.

التشغيل/الإطفاء بيتحترم وقت التشغيل: التطبيق بيقرا /api/packages/installed، ولو باكيج الريلز مطفي بيتخفّي ما يضيفه من واجهة.
