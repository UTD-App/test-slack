Gifts Package

The Gifts package (utd/gifts) adds a complete in-app gifting experience: a managed gift catalog, a send/picker flow, gift history, and sender/receiver levels driven by EXP. It is the engine that turns coins into diamonds — it spends the sender's coins (via the Wallet package) and credits the receiver's diamonds, recording a gift log for every send.

What it does

Installing the package binds the Base App\Contracts\GiftSender and App\Contracts\GiftDirectory contracts. Once bound, gifting "lights up" everywhere it is wired (Moment now; Room / Live / Reels later) without those host features needing a hard dependency on Gifts. Each send: validates the gift, spends coins, credits diamonds, writes a gift_logs row per receiver, bumps the gift's use count, and fires an App\Events\Gifts\GiftSent event so other packages can layer their own effects.

Key features

  • Gift catalog — admin-managed gifts and categories (Filament resources), with media (image / SVGA / mp4), price in coins, and a cached read API (GET /api/gifts, GET /api/gifts/categories).
  • Send & picker — single-receiver and multi-receiver (batch) sends through the GiftSender contract; one debit, one batch_id, a gift log + event per receiver. The Flutter picker sheet opens from any host feature via the GiftBridge seam.
  • History — sent / received gift history per user (GET /api/gifts/history?type=sent|received) plus per-context gifters and receivers (GET /api/gifts/context/{type}/{id}).
  • Levels & EXP — every send banks EXP (sender from coins spent, receiver from diamonds earned); the level is the highest gift_levels threshold the stored EXP reaches. Admin-tunable EXP rates on the Gift EXP settings page.
  • Seams for other packages — GiftSender and GiftDirectory are bound here; optional contracts (LuckyGiftResolver, GiftBagProvider, VipLevelProvider) stay graceful no-ops until their own plugins bind them. The GiftSent event lets Room / Agency / Family / Levels packages add their splits and effects.
  • Profile contribution — contributes a "received gifts" section (gift wall + sender/receiver level badges + top supporters) to the Profile package, and "Gifts received" / "Gifts sent" tabs to the admin User profile.

Dependencies

Requires the Wallet package: the sender spends coins and the receiver earns diamonds. The package registers the diamonds earning currency on the Wallet at runtime, so no Wallet edits are needed.

العربية

حزمة الهدايا

تضيف حزمة الهدايا (utd/gifts) تجربة إهداء متكاملة داخل التطبيق: كتالوج هدايا يُدار من لوحة التحكم، وشاشة اختيار وإرسال، وسجلّ للهدايا، ومستويات للمُرسِل والمستلِم تعتمد على نقاط الخبرة. وهي المحرّك الذي يحوّل العملات إلى ألماس — يخصم coins من رصيد المُرسِل (عبر حزمة المحفظة) ويضيف diamonds لرصيد المستلِم، مع تسجيل كل عملية إهداء.

ماذا تفعل الحزمة

عند التثبيت تربط الحزمة عقدَي القاعدة App\Contracts\GiftSender و App\Contracts\GiftDirectory. وبمجرد الربط تعمل ميزة الإهداء في كل مكان موصول بها (مومنت الآن؛ والغرف والبث والريلز لاحقًا) دون أن تعتمد تلك الميزات على حزمة الهدايا اعتمادًا صلبًا. كل عملية إرسال: تتحقّق من الهدية، تخصم العملات، تضيف الألماس، تكتب صفًّا في gift_logs لكل مستلِم، تزيد عدّاد استخدام الهدية، وتطلق الحدث App\Events\Gifts\GiftSent لتبني عليه الحزم الأخرى تأثيراتها.

أهم المزايا

  • كتالوج الهدايا — هدايا وفئات تُدار من لوحة التحكم (موارد Filament)، مع وسائط (صورة / SVGA / mp4)، وسعر بالعملات، وواجهة قراءة مع تخزين مؤقت (GET /api/gifts، GET /api/gifts/categories).
  • الإرسال والاختيار — إرسال لمستلِم واحد أو لعدة مستلِمين (دُفعة) عبر عقد GiftSender؛ خصم واحد، وbatch_id واحد، وسجلّ وحدث لكل مستلِم. تُفتح شاشة الاختيار في فلاتر من أي ميزة مضيفة عبر وسيط GiftBridge.
  • السجلّ — سجلّ الهدايا المُرسلة / المستلَمة لكل مستخدم (GET /api/gifts/history?type=sent|received) إضافةً إلى المُهدين والمستلِمين حسب السياق (GET /api/gifts/context/{type}/{id}).
  • المستويات ونقاط الخبرة — كل إرسال يبنك نقاط خبرة (المُرسِل من العملات المنفقة، والمستلِم من الألماس المكتسب)؛ والمستوى هو أعلى عتبة في gift_levels تصلها النقاط المخزّنة. ومعدّلات الخبرة قابلة للضبط من صفحة إعدادات الخبرة.
  • وصلات للحزم الأخرى — يُربط هنا GiftSender و GiftDirectory؛ بينما تبقى العقود الاختيارية (LuckyGiftResolver، GiftBagProvider، VipLevelProvider) بلا تأثير بأمان حتى تربطها إضافاتها. ويتيح حدث GiftSent لحزم الغرف والوكالات والعائلة والمستويات إضافة حصصها وتأثيراتها.
  • المساهمة في الملف الشخصي — تضيف قسم "الهدايا المستلَمة" (حائط الهدايا + شارات مستوى المُرسِل/المستلِم + أبرز الداعمين) إلى حزمة الملف الشخصي، وتبويبَي "الهدايا المستلَمة" / "الهدايا المُرسلة" في ملف المستخدم بلوحة التحكم.

المتطلبات

تتطلّب حزمة المحفظة: يخصم المُرسِل coins ويكسب المستلِم diamonds. وتسجّل الحزمة عملة الكسب diamonds على المحفظة وقت التشغيل، فلا حاجة لتعديل المحفظة.
