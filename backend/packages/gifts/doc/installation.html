Installing the Gifts Package

The Gifts package has two parts: a backend Laravel package (utd/gifts) and a Flutter package (gifts). It depends on the Wallet package — install Wallet first.

1. Backend package

  1. Add the package to the Base project. It lives at packages/gifts; reference it as a path repository in the root composer.json: 
    "repositories": [
  { "type": "path", "url": "packages/gifts" }
],
"require": {
  "utd/gifts": "*"
}
  2. Install / refresh autoload: 
    composer require utd/gifts
# or, if already present as a path repo:
composer update utd/gifts
composer dump-autoload
    The service provider Utd\Gifts\Providers\GiftsServiceProvider is auto-discovered via the package's extra.laravel.providers — no manual registration in config/app.php is required.
  3. Run the migrations (gift categories, gifts, gift logs, gift levels, user EXP, settings): 
    php artisan migrate
  4. (Optional) seed a starter catalog: 
    php artisan db:seed --class="Utd\Gifts\Database\Seeders\GiftCatalogSeeder"
  5. Register the Filament plugin in app/Providers/Filament/AdminPanelProvider.php (this part is manual — only the service provider is auto-discovered): 
    ->plugin(\Utd\Gifts\Filament\GiftsPlugin::make())
  6. Enable the package in admin → Packages. While disabled, the provider does not bind GiftSender and the gift routes return 403 (package.enabled:gifts).
Wallet currency: the sender spends coins and the receiver earns diamonds. The provider auto-registers diamonds as a Wallet earning currency at runtime, so no Wallet config edit is needed. Confirm Wallet is installed and enabled.

2. Flutter package

  1. Add the path dependency in the app's pubspec.yaml: 
    dependencies:
  gifts:
    path: packages/gifts
    then run flutter pub get.
  2. Register the feature where the app builds its feature list: 
    import 'package:gifts/gifts.dart';

// in the app's feature registry
GiftsFeature(),
    Registering GiftsFeature wires the GiftBridge seam so host features (Moment, Reels…) can open the gift picker, adds the Gifts history drawer entry, the profile gifts section, the gift routes, and the bundled translations.
GiftsFeature self-checks the backend at startup: if no Wallet package is reported it does not register the GiftBridge, so host features hide their gift button. It auto-enables on the next launch once Wallet is installed.

3. Verify

Hit GET /api/gifts/categories and GET /api/gifts (authenticated) — both should return data once the catalog is seeded. In the app, the gift button should appear on a Moment, and sending a gift should debit coins and credit diamonds.

العربية

تثبيت حزمة الهدايا

تتكوّن حزمة الهدايا من جزأين: حزمة الخلفية في لارافيل (utd/gifts) وحزمة فلاتر (gifts). وهي تعتمد على حزمة المحفظة — ثبّت المحفظة أولًا.

١. حزمة الخلفية

  1. أضف الحزمة إلى مشروع القاعدة. مكانها packages/gifts؛ أشِر إليها كمستودع مسار في composer.json الجذري: 
    "repositories": [
  { "type": "path", "url": "packages/gifts" }
],
"require": {
  "utd/gifts": "*"
}
  2. ثبّت / حدّث التحميل التلقائي: 
    composer require utd/gifts
# أو إن كانت مضافة مسبقًا كمستودع مسار:
composer update utd/gifts
composer dump-autoload
    يتم اكتشاف مزوّد الخدمة Utd\Gifts\Providers\GiftsServiceProvider تلقائيًا عبر extra.laravel.providers — فلا حاجة لتسجيله يدويًا في config/app.php.
  3. شغّل الترحيلات (فئات الهدايا، الهدايا، سجلّات الهدايا، المستويات، خبرة المستخدم، الإعدادات): 
    php artisan migrate
  4. (اختياري) ازرع كتالوجًا مبدئيًا: 
    php artisan db:seed --class="Utd\Gifts\Database\Seeders\GiftCatalogSeeder"
  5. سجّل إضافة Filament في app/Providers/Filament/AdminPanelProvider.php (هذا الجزء يدوي — المُكتشف تلقائيًا هو مزوّد الخدمة فقط): 
    ->plugin(\Utd\Gifts\Filament\GiftsPlugin::make())
  6. فعّل الحزمة من لوحة التحكم ← الحزم. وأثناء التعطيل لا يربط المزوّد GiftSender وتُعيد مسارات الهدايا الرمز 403 (package.enabled:gifts).
عملة المحفظة: يخصم المُرسِل coins ويكسب المستلِم diamonds. يسجّل المزوّد diamonds تلقائيًا كعملة كسب على المحفظة وقت التشغيل، فلا حاجة لتعديل إعدادات المحفظة. تأكّد أن المحفظة مثبّتة ومفعّلة.

٢. حزمة فلاتر

  1. أضف اعتماد المسار في pubspec.yaml الخاص بالتطبيق: 
    dependencies:
  gifts:
    path: packages/gifts
    ثم نفّذ flutter pub get.
  2. سجّل الميزة حيث يبني التطبيق قائمة ميزاته: 
    import 'package:gifts/gifts.dart';

// في سجلّ ميزات التطبيق
GiftsFeature(),
    تسجيل GiftsFeature يربط وسيط GiftBridge لتفتح الميزات المضيفة (مومنت، ريلز…) شاشة اختيار الهدايا، ويضيف مدخل سجلّ الهدايا في القائمة الجانبية، وقسم الهدايا في الملف الشخصي، ومسارات الهدايا، والترجمات المضمّنة.
تتحقّق GiftsFeature من الخلفية عند الإقلاع: إن لم تكن المحفظة مُبلّغة فإنها لا تسجّل GiftBridge، فتُخفي الميزات المضيفة زر الهدية. وتُفعَّل تلقائيًا في الإقلاع التالي بمجرد تثبيت المحفظة.

٣. التحقّق

استدعِ GET /api/gifts/categories وGET /api/gifts (بمصادقة) — يجب أن يعيدا بيانات بعد زرع الكتالوج. وفي التطبيق يجب أن يظهر زر الهدية على المومنت، وعند إرسال هدية يُخصم رصيد العملات ويُضاف الألماس.
