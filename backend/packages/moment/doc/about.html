UTD Package · utd/moment

Moment — the social moments feed

Posts · media galleries · reactions · threaded comments · reports · gifting

About (English)

Moment is a standalone, drop-in package (utd/moment, namespace Utd\Moment) that adds a complete social moments feed on top of the UTD base — without touching the base. Users post moments (text + image gallery), react to them, write threaded comments and replies, report offending content, and send gifts. It self-registers in the base PackageRegistry and can be toggled on/off from admin/packages.

Features

  • Feed — paginated moments feed with multiple list types: a user's own posts (type 1), the viewer's liked moments (type 2), a Following feed (type 3, via FollowProvider), the full feed (types 4/5), and a follow variant (type 6). Eagle-style ordering floats not-yet-liked moments to the top, then most-recent first.
  • Posts & media — each moment has a text description and a multi-image gallery (moment_galleries); up to 9 images per post, validated for MIME type and size, uploaded via the base MediaUploader contract and resolved to absolute URLs.
  • Likes & reactions — like a moment with a reaction type (like / love / haha / wow / sad / angry); the feed pre-computes the viewer's reaction and a per-type breakdown per page (no N+1).
  • Comments & replies — threaded comments with replies (parent_id) and likes on the comments themselves.
  • Reports — report a moment or a comment with a reason and description (spam / abuse / nudity / violence / other), surfaced for moderation in the Filament admin.
  • Gifting — sending/listing gifts on a moment auto-enables when the Gifts package binds App\Contracts\GiftDirectory; until then gift routes return a safe 503 and the rest of the feed keeps working. Gift counts and coin totals are read through the GiftDirectory seam — no hard dependency on Gifts.
  • Following feed — the Following feed filters moments by the social follow graph through the optional App\Contracts\FollowProvider seam; when the social package isn't installed it gracefully falls back to the full feed.
  • Profile & admin integration — contributes a profile tab via ProfileContributorRegistry, plus "Reports filed" and "User moments" tabs on the admin user page via UserProfileTabRegistry, and a MomentFeed Filament moderation page.

API surface

All routes sit under auth:sanctum + ban/presence/localization middleware + package.enabled:moment (returns 403 while the package is disabled):

GET    /api/moment                 # feed (type=1..6)
POST   /api/moment                 # create a moment (text + multi_image[])
GET    /api/moment/{id}            # show one moment
DELETE /api/moment/{id}            # delete (owner only)
GET    /api/moment/user/{user_id}  # a specific user's moments

GET    /api/moment/{id}/comment            # comments
POST   /api/moment/{id}/comment            # add a comment / reply
POST   /api/moment/{id}/comment/{cid}/like   # react to a comment
POST   /api/moment/{id}/comment/{cid}/report # report a comment
DELETE /api/moment/{id}/comment/{cid}

GET    /api/moment/{id}/like       # who liked
POST   /api/moment/{id}/like       # like / unlike (+ reaction_type)
POST   /api/moment/{id}/report     # report a moment

POST   /api/moment/{id}/gift           # send a gift (Gifts pkg; else 503)
GET    /api/moments/{id}/gifts         # gifts on a moment
GET    /api/moments/users/{id}/gifts   # gifts a user sent

الشرح (بالعربية)

Moment حزمة مستقلة (drop-in) باسم utd/moment و namespace Utd\Moment، بتضيف فيد لحظات اجتماعي كامل فوق قاعدة UTD — من غير ما تلمس القاعدة. المستخدم ينشر لحظة (نص + معرض صور), يتفاعل عليها, يكتب تعليقات وردود متداخلة, يبلّغ عن المحتوى المخالف, ويبعت هدايا. الحزمة بتسجّل نفسها في PackageRegistry وتقدر تشغّلها/تطفّيها من admin/packages.

المميزات

  • الفيد — فيد لحظات بصفحات وأنواع متعددة: منشورات مستخدم معيّن (type 1), اللحظات اللي عملها المستخدم لايك (type 2), فيد المتابَعين (type 3 عبر FollowProvider), الفيد الكامل (type 4/5), ونسخة متابعة (type 6). الترتيب على طريقة Eagle: اللحظات اللي لسه ما اتعملهاش لايك بتطفو لفوق, بعدين الأحدث.
  • المنشورات والوسائط — كل لحظة فيها وصف نصّي ومعرض صور متعدد (moment_galleries) لحد 9 صور للمنشور, متحقّق من نوع الملف وحجمه, بترفع عبر عقد القاعدة MediaUploader وبتترجم لروابط مطلقة.
  • الإعجابات والتفاعلات — لايك على اللحظة مع نوع تفاعل (like / love / haha / wow / sad / angry); الفيد بيحسب تفاعل المشاهد وتوزيع الأنواع لكل صفحة مسبقاً (من غير N+1).
  • التعليقات والردود — تعليقات مترابطة بردود متداخلة (parent_id) وإعجاب على التعليقات نفسها.
  • البلاغات — إبلاغ عن لحظة أو تعليق بسبب ووصف (spam / abuse / nudity / violence / other) بتظهر للإشراف في لوحة Filament.
  • الهدايا — إرسال/عرض الهدايا على اللحظة بيتفعّل أوتوماتيك لما حزمة Gifts تربط App\Contracts\GiftDirectory; لحد ما يحصل ده مسارات الهدايا بترجع 503 بشكل آمن وباقي الفيد شغّال. العدّادات وإجمالي الكوينز بتتقري من خلال seam الـ GiftDirectory — من غير اعتماد صارم على Gifts.
  • فيد المتابَعين — فيد المتابَعين بيفلتر اللحظات حسب شبكة المتابعة عبر seam اختياري App\Contracts\FollowProvider; لو حزمة السوشيال مش متثبّتة بيرجع للفيد الكامل بأمان.
  • التكامل مع البروفايل واللوحة — بيضيف تبويب على البروفايل عبر ProfileContributorRegistry, وتبويبات "البلاغات المُقدّمة" و"لحظات المستخدم" على صفحة المستخدم في اللوحة عبر UserProfileTabRegistry, وصفحة إشراف MomentFeed في Filament.
