Wallet Package

The Wallet package (utd/wallet) owns the user's in-app COIN wallet: a per-user balance plus one unified, append-only wallet_transactions ledger that records every coin movement. It is the single path through which coins are credited or debited — admin charges, gateway recharges, gifts, games and rewards all land here.

Scope — coins only. The dollar / earnings wallet and withdrawals live in the target package; diamonds in the agency / gifts packages. A dollar→coins conversion is recorded here as a wallet_transactions row of type exchange. The currency list is config-driven, so an extra currency can be added with no schema change.

What it does

  • Binds the Base contract. The provider binds App\Contracts\WalletContract to its DatabaseWallet implementation (or to the Base NullWallet when the package is disabled — reads return 0, writes throw). The Base only ships the contract, facade, events and the Null seam; this package is the real engine.
  • bcmath decimal balances. Balances are decimal(20,2) and all money math is done with bcmath on the decimal strings, so there is no float rounding drift. Every move is row-locked (FOR UPDATE) and wrapped in a DB transaction; available = balance − held (held = reserved for pending withdrawals).
  • Idempotent ledger. A caller that retries the same money op (payment webhook, gift, exchange) passes meta['idempotency_key']; the existing ledger row is replayed instead of moving balance twice, and the unique-key race between concurrent identical calls is resolved safely. Every credit/debit fires WalletCredited / WalletDebited.
  • Charges (admin). ChargeService is the single entry point for manual charges (admin → user today; agencies / area managers later through the same method). It moves balance via the Wallet and records a charges history row linked to the ledger entry. The Filament Charges resource exposes a list + create form.
  • Recharge catalogue. Coin (coin packages buyable for money) and PaymentCoin (payment-method groups) model the recharge screen, with a coin_logs gateway-purchase log. They are served to the app via GET /api/coins and GET /api/coins/payment-methods and managed from the new Filament Coins / Payment Coins admin resources.
  • Transaction types. The CoinTransactionType enum (ported from Eagle's UserCoinLogType) standardises wallet_transactions.type across ~45 flows (admin/area-manager/BD charges, payment / Google Pay / Huawei Pay, exchange, gifts & rooms, games, rewards, special items). Each value has a localized label (wallet.tx_type.*) served to the app.

API (authenticated)

  • GET /api/wallet/balances — all balances for the current user.
  • GET /api/wallet/transactions?currency=&type=&start_date=&end_date=&page= — the paginated ledger (newest first); the envelope carries pagination meta (has_more).
  • GET /api/coins — coin recharge packages.
  • GET /api/coins/payment-methods — payment-method groups.

Admin (Filament — group "Wallet")

  • Charges — list + create a manual charge.
  • Wallet Transactions — the read-only ledger.
  • Coins / Payment Coins — manage the recharge catalogue.

Data model

Migrations: user_wallets, wallet_transactions (+ idempotency key), coins, payment_coins, coin_logs, charges.

العربية

حزمة المحفظة

تملك حزمة المحفظة (utd/wallet) محفظة الكوينز داخل التطبيق: رصيد لكل مستخدم بالإضافة إلى سجلّ موحّد لا يُعدَّل (wallet_transactions) يسجّل كل حركة كوينز. وهي المسار الوحيد الذي تُضاف أو تُخصم عبره الكوينز — شحنات الإدارة، وشحن البوابات، والهدايا، والألعاب، والمكافآت كلها تمر هنا.

النطاق — الكوينز فقط. محفظة الدولار / الأرباح والسحوبات موجودة في حزمة target؛ والألماس في حزمتي agency / gifts. ويُسجَّل تحويل الدولار إلى كوينز هنا كصفّ في wallet_transactions من النوع exchange. قائمة العملات قابلة للتهيئة، فيمكن إضافة عملة دون أي تغيير في المخطط.

ماذا تفعل

  • تربط عقد القاعدة. يربط المزوّد App\Contracts\WalletContract بتنفيذها DatabaseWallet (أو بـ NullWallet من القاعدة عند تعطيل الحزمة — تُعيد القراءة 0 وتُطلِق الكتابة استثناء). القاعدة تشحن العقد والـ facade والأحداث وبذرة Null فقط؛ وهذه الحزمة هي المحرّك الفعلي.
  • أرصدة عشرية بـ bcmath. الأرصدة من نوع decimal(20,2) وكل حسابات المال تُجرى بـ bcmath على السلاسل العشرية، فلا يوجد انحراف تقريب للأعداد العائمة. وكل حركة مقفولة على الصف (FOR UPDATE) ضمن معاملة قاعدة بيانات؛ المتاح = الرصيد − المحجوز (المحجوز = محتجز لسحوبات معلّقة).
  • سجلّ غير قابل للتكرار (idempotent). المستدعي الذي يعيد المحاولة لنفس العملية المالية (ويب-هوك دفع، هدية، تحويل) يمرّر meta['idempotency_key']؛ فيُعاد تشغيل صفّ السجلّ القائم بدل تحريك الرصيد مرتين، ويُحلّ تسابق المفتاح الفريد بين النداءات المتطابقة المتزامنة بأمان. وكل إضافة/خصم يُطلِق WalletCredited / WalletDebited.
  • الشحنات (الإدارة). ChargeService هو نقطة الدخول الوحيدة للشحنات اليدوية (الإدارة ← المستخدم اليوم؛ والوكالات / مدراء المناطق لاحقًا عبر نفس الطريقة). يحرّك الرصيد عبر المحفظة ويسجّل صفّ تاريخ في charges مرتبطًا بقيد السجلّ. ويوفّر مورد Filament الشحنات قائمة + نموذج إنشاء.
  • كتالوج الشحن. يمثّل Coin (باقات الكوينز القابلة للشراء بالمال) وPaymentCoin (مجموعات طرق الدفع) شاشة الشحن، مع سجلّ شراء البوابات coin_logs. وتُقدَّم للتطبيق عبر GET /api/coins و GET /api/coins/payment-methods وتُدار من موارد Filament الجديدة الكوينز / كوينز الدفع.
  • أنواع المعاملات. يوحّد التعداد CoinTransactionType (المنقول من UserCoinLogType في Eagle) قيمة wallet_transactions.type عبر ~45 تدفّقًا (شحنات الإدارة / مدير المنطقة / BD، الدفع / Google Pay / Huawei Pay، التحويل، الهدايا والغرف، الألعاب، المكافآت، العناصر الخاصة). ولكل قيمة تسمية مُترجمة (wallet.tx_type.*) تُقدَّم للتطبيق.

واجهة البرمجة (بمصادقة)

  • GET /api/wallet/balances — كل أرصدة المستخدم الحالي.
  • GET /api/wallet/transactions?currency=&type=&start_date=&end_date=&page= — السجلّ المقسّم لصفحات (الأحدث أولًا)؛ يحمل الغلاف بيانات تقسيم meta (has_more).
  • GET /api/coins — باقات شحن الكوينز.
  • GET /api/coins/payment-methods — مجموعات طرق الدفع.

لوحة التحكم (Filament — مجموعة "المحفظة")

  • الشحنات — قائمة + إنشاء شحنة يدوية.
  • معاملات المحفظة — السجلّ للقراءة فقط.
  • الكوينز / كوينز الدفع — إدارة كتالوج الشحن.

نموذج البيانات

الترحيلات: user_wallets، wallet_transactions (+ مفتاح idempotency)، coins، payment_coins، coin_logs، charges.
