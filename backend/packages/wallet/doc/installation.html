Installing the Wallet Package

The Wallet package has two parts: a backend Laravel package (utd/wallet) and a Flutter package (wallet). It depends only on the Base seam (contract / facade / events / NullWallet), which already ships in the Base project — no other package is required.

1. Backend package

  1. Add the package to the Base project. It lives at packages/wallet; reference it as a path repository in the root composer.json and register the PSR-4 autoload: 
    "repositories": [
  { "type": "path", "url": "packages/wallet" }
],
"require": {
  "utd/wallet": "*"
},
"autoload": {
  "psr-4": { "Utd\\Wallet\\": "packages/wallet/src/" }
},
"autoload-dev": {
  "psr-4": { "Utd\\Wallet\\Tests\\": "packages/wallet/tests/" }
}
  2. Install / refresh autoload: 
    composer require utd/wallet
# or, if already present as a path repo:
composer update utd/wallet
composer dump-autoload
    The service provider Utd\Wallet\Providers\WalletServiceProvider is auto-discovered via the package's extra.laravel.providers — no manual entry in config/app.php is required. It binds the Base WalletContract to DatabaseWallet (or NullWallet when disabled) and wires config, migrations, routes and translations.
  3. Run the migrations (user_wallets, wallet_transactions, coins, payment_coins, coin_logs, charges): 
    php artisan migrate
  4. Register the Filament plugin in app/Providers/Filament/AdminPanelProvider.php so the Charges, Wallet Transactions, Coins and Payment Coins resources appear under the "Wallet" group: 
    ->plugin(\Utd\Wallet\Filament\WalletPlugin::make())
  5. Enable the package in admin → Packages. While disabled (WALLET_ENABLED=false or toggled off), the provider binds NullWallet: balance reads return 0 and writes throw.
Config: config/config.php (merged as wallet) sets enabled, default_currency = coins and currencies = ['coins']. Add a string to currencies to support an extra currency — DatabaseWallet serves it with no schema change.

2. Flutter package

  1. Add the path dependency in the app's pubspec.yaml: 
    dependencies:
  wallet:
    path: packages/wallet
    then run flutter pub get.
  2. Register the feature where the app builds its feature list: 
    import 'package:wallet/wallet.dart';

// in the app's feature registry
WalletFeature(),
    Registering WalletFeature wires the wallet cubit/repository, adds the coin balance card to the user's own profile (UiSlot.userProfile) and a drawer entry, the /wallet route, the bundled wallet.* translations, and the UTD Studio coin-card source. It is hidden automatically when the backend wallet package is disabled (packageSlug: 'wallet').

3. Verify

Hit GET /api/wallet/balances and GET /api/wallet/transactions (authenticated) — both should return the standard envelope. In the app, the coin card should appear on your own profile and open the wallet page with the (paginated) transaction history.

العربية

تثبيت حزمة المحفظة

تتكوّن حزمة المحفظة من جزأين: حزمة الخلفية في لارافيل (utd/wallet) وحزمة فلاتر (wallet). وهي تعتمد فقط على بذرة القاعدة (العقد / الـ facade / الأحداث / NullWallet) الموجودة أصلًا في مشروع القاعدة — فلا حاجة لأي حزمة أخرى.

١. حزمة الخلفية

  1. أضف الحزمة إلى مشروع القاعدة. مكانها packages/wallet؛ أشِر إليها كمستودع مسار في composer.json الجذري وسجّل تحميل PSR-4: 
    "repositories": [
  { "type": "path", "url": "packages/wallet" }
],
"require": {
  "utd/wallet": "*"
},
"autoload": {
  "psr-4": { "Utd\\Wallet\\": "packages/wallet/src/" }
},
"autoload-dev": {
  "psr-4": { "Utd\\Wallet\\Tests\\": "packages/wallet/tests/" }
}
  2. ثبّت / حدّث التحميل التلقائي: 
    composer require utd/wallet
# أو إن كانت مضافة مسبقًا كمستودع مسار:
composer update utd/wallet
composer dump-autoload
    يتم اكتشاف مزوّد الخدمة Utd\Wallet\Providers\WalletServiceProvider تلقائيًا عبر extra.laravel.providers — فلا حاجة لإدخاله يدويًا في config/app.php. وهو يربط WalletContract من القاعدة بـ DatabaseWallet (أو NullWallet عند التعطيل) ويهيّئ الإعدادات والترحيلات والمسارات والترجمات.
  3. شغّل الترحيلات (user_wallets، wallet_transactions، coins، payment_coins، coin_logs، charges): 
    php artisan migrate
  4. سجّل إضافة Filament في app/Providers/Filament/AdminPanelProvider.php لتظهر موارد الشحنات ومعاملات المحفظة والكوينز وكوينز الدفع ضمن مجموعة "المحفظة": 
    ->plugin(\Utd\Wallet\Filament\WalletPlugin::make())
  5. فعّل الحزمة من لوحة التحكم ← الحزم. وأثناء التعطيل (WALLET_ENABLED=false أو الإيقاف من اللوحة) يربط المزوّد NullWallet: تُعيد قراءة الرصيد 0 وتُطلِق الكتابة استثناء.
الإعداد: يضبط config/config.php (المدموج باسم wallet) القيم enabled وdefault_currency = coins و currencies = ['coins']. أضف سلسلة إلى currencies لدعم عملة إضافية — ويقدّمها DatabaseWallet دون أي تغيير في المخطط.

٢. حزمة فلاتر

  1. أضف اعتماد المسار في pubspec.yaml الخاص بالتطبيق: 
    dependencies:
  wallet:
    path: packages/wallet
    ثم نفّذ flutter pub get.
  2. سجّل الميزة حيث يبني التطبيق قائمة ميزاته: 
    import 'package:wallet/wallet.dart';

// في سجلّ ميزات التطبيق
WalletFeature(),
    تسجيل WalletFeature يربط مكعّب/مستودع المحفظة، ويضيف بطاقة رصيد الكوينز إلى الملف الشخصي للمستخدم نفسه (UiSlot.userProfile) ومدخلًا في القائمة الجانبية، ومسار /wallet، وترجمات wallet.* المضمّنة، ومصدر بطاقة الكوينز في UTD Studio. وتُخفى تلقائيًا عند تعطيل حزمة wallet في الخلفية (packageSlug: 'wallet').

٣. التحقّق

استدعِ GET /api/wallet/balances وGET /api/wallet/transactions (بمصادقة) — يجب أن يعيدا الغلاف القياسي. وفي التطبيق يجب أن تظهر بطاقة الكوينز على ملفك الشخصي وتفتح صفحة المحفظة مع تاريخ المعاملات (المقسّم لصفحات).
